在当前工作目录下创建一个极简用户管理后端，使用 Node 24 内置的 node:sqlite（不需要 better-sqlite3，也不需要任何 native 依赖）。

环境：
- Node 版本：v24.15.0（已确认 node:sqlite 可用，命令：`node -e "require('node:sqlite')"`）
- 平台：Windows
- 工作目录：当前 shell 的 cwd

具体步骤：

1. 用 bash 工具创建子目录 backend/
2. 用 write 工具创建 backend/package.json（只声明 "type": "module"，不需要任何 dependencies！）：
   {"name":"backend","version":"1.0.0","type":"module","main":"server.js"}
3. 用 write 工具创建 backend/db.js，使用 node:sqlite：
   - 打开 backend/users.db（DatabaseSync）
   - 建表 IF NOT EXISTS users (id INTEGER PRIMARY KEY AUTOINCREMENT, name TEXT NOT NULL, email TEXT NOT NULL UNIQUE, role TEXT DEFAULT 'user', created_at TEXT DEFAULT CURRENT_TIMESTAMP)
   - 导出 db 实例和 ensure_schema() 函数
4. 用 write 工具创建 backend/server.js（CommonJS 也行，但 module 更现代 — 用 ESM 因为 package.json 有 "type":"module"）：
   - listen 127.0.0.1:4400
   - 实现 REST：
     * GET /api/users        → {users:[...]}
     * GET /api/users/:id    → 单个或 404
     * POST /api/users       body {name,email,role?} → 新用户
     * PUT /api/users/:id    body {name?,email?,role?} → 更新后用户
     * DELETE /api/users/:id → {deleted: id}
   - 用 prepared statements（db.prepare('SELECT * FROM users').all() 等）
   - 加 CORS：Access-Control-Allow-Origin: http://localhost:5500

5. 用 bash 工具后台启动：cd backend && node server.js > server.log 2>&1 &
6. sleep 3
7. 用 bash 跑：curl http://127.0.0.1:4400/api/users
8. 用 bash 跑：curl -X POST http://127.0.0.1:4400/api/users -H 'Content-Type: application/json' -d '{"name":"Alice","email":"alice@example.com"}'
9. 再 curl 一次 GET /api/users，确认 Alice 出现
