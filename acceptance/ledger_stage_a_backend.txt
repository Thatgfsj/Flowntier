在当前工作目录下创建一个完整的家庭记账本后端，使用 Node 24 内置的 node:sqlite（零依赖，零编译）。

环境：
- Node v24.15.0（已确认 `node:sqlite` 可用）
- 平台：Windows
- 后端监听 127.0.0.1:4401
- 前端会用 python http.server 在 5501
- CORS：允许 http://localhost:5501

具体步骤：

1. 用 bash 创建子目录 backend/

2. 用 write 创建 backend/package.json：
   {"name":"ledger-backend","version":"1.0.0","type":"module","main":"server.js"}
   （不需要任何 dependencies！）

3. 用 write 创建 backend/db.js（用 node:sqlite）：
   - 打开 backend/ledger.db
   - 三个表 IF NOT EXISTS:
     - users(id INTEGER PRIMARY KEY AUTOINCREMENT, name TEXT NOT NULL, role TEXT DEFAULT 'member', created_at TEXT DEFAULT CURRENT_TIMESTAMP)
     - accounts(id INTEGER PRIMARY KEY AUTOINCREMENT, user_id INTEGER NOT NULL, name TEXT NOT NULL, kind TEXT DEFAULT 'cash', balance REAL DEFAULT 0, created_at TEXT DEFAULT CURRENT_TIMESTAMP, FOREIGN KEY(user_id) REFERENCES users(id))
     - transactions(id INTEGER PRIMARY KEY AUTOINCREMENT, account_id INTEGER NOT NULL, amount REAL NOT NULL, category TEXT NOT NULL, note TEXT DEFAULT '', occurred_at TEXT DEFAULT CURRENT_TIMESTAMP, created_at TEXT DEFAULT CURRENT_TIMESTAMP, FOREIGN KEY(account_id) REFERENCES accounts(id))
   - 导出 db 和 ensure_schema() 函数
   - 准备 5 个常用 prepared statements 并导出

4. 用 write 创建 backend/server.js：
   - listen 127.0.0.1:4401
   - 用原生 http 模块（不要 npm install）
   - 路由：
     * GET    /api/users                     → 列表
     * POST   /api/users                     body: {name, role?}
     * GET    /api/users/:id/accounts        → 用户的所有账户
     * POST   /api/accounts                  body: {user_id, name, kind?, balance?}
     * POST   /api/transactions              body: {account_id, amount, category, note?, occurred_at?}
     * GET    /api/accounts/:id/transactions  → 某账户的所有流水 (limit 200)
     * DELETE /api/transactions/:id           删除
     * GET    /api/report/summary?user_id=N  →  {total_in, total_out, balance, by_category: {food: 1230, ...}}
   - 全部用 prepared statements
   - CORS: Access-Control-Allow-Origin: http://localhost:5501
   - preflight OPTIONS 也要正确返回
   - JSON 解析错误返回 400 + {error: msg}
   - 找不到返回 404 + {error: "not found"}

5. 用 write 创建 backend/seed.js（可选但建议）：
   - 创建 1 个 user "张三"，2 个 accounts (现金 + 银行卡)，3 笔 transactions

6. 用 bash 启动：cd backend && node server.js > server.log 2>&1 &

7. sleep 3 秒

8. 用 bash 跑：curl http://127.0.0.1:4401/api/users 应该返回用户列表

9. 创建一个 transaction 验证 POST：curl -X POST .../api/transactions -d '{...}'
