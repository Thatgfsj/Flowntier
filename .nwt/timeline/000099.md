# 000099 — Cargo workspace version 同步到 0.4.22

## 主席反馈
"版本号每次只加0.01, 改回来" — Cargo workspace version 卡在 0.4.0,
但 package.json / tauri.conf.json / setup.exe 文件名都已经是 0.4.22。

## 调查

3 个地方声明版本号:

| 位置 | 旧值 | 来源 |
|------|------|------|
| `Cargo.toml` `[workspace.package].version` | 0.4.0 | workspace 根 |
| `apps/desktop/src-tauri/Cargo.toml` `[package].version` | **0.4.0 (写死)** | 显式覆盖 |
| `apps/desktop/package.json` | 0.4.22 | Node |
| `apps/desktop/src-tauri/tauri.conf.json` | 0.4.22 | Tauri config |

6 个子 crate (`tauri-core`, `event-bus`, `agent-core`, `pipe-server`,
`config`, `storage`) 都用 `version = { workspace = true }` 正确继承。

唯一不一致: `flowntier-desktop` 的 Cargo.toml 没继承 workspace,
硬编码 "0.4.0"。这导致:
- `cargo build` 产物 metadata 报 0.4.0
- sidecar 内嵌版本字符串 = 0.4.0
- 但 setup.exe 文件名 = 0.4.22 (Tauri 从 tauri.conf.json 读)
- chairman 看到两个版本号不一致 (000099 之前的 cargo metadata 输出)

## 修复

1. `Cargo.toml` workspace.package.version: 0.4.0 → 0.4.22
2. `apps/desktop/src-tauri/Cargo.toml` [package].version:
   `"0.4.0"` → `{ workspace = true }`

`cargo metadata` 验证 7 个 crate 全部 0.4.22:
```
tauri-core       0.4.22
config           0.4.22
event-bus        0.4.22
storage          0.4.22
agent-core       0.4.22
pipe-server      0.4.22
flowntier-desktop 0.4.22
```

## 后续规则

今后每次事件交付:
1. 改 `[workspace.package].version` 一处即可
2. 所有子 crate 自动同步 (workspace 继承)
3. package.json / tauri.conf.json 手动同步 (Tauri 不读 Cargo.toml
   的 version, 反而读它自己的 config)
4. setup.exe 文件名由 Tauri 从 tauri.conf.json 读, 也是 0.4.22

版本号步长: 每次事件 +0.01 (v0.4.22 → v0.4.23 → v0.4.24 ...)。
不跨 minor (0.4 → 0.5), 不跨 major (0 → 1)。
