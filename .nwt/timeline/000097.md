# 000097 — v3 NSIS taskkill belt + `/api/router/models` 空态修复

## 主席反馈（接续 000095/000096）
1. v0.4.22 setup.exe 安装升级时仍报
   "无法打开要写入的文件…O:\Flowntier\flowntier_runtime.exe"
   → event 000093 的 v2 taskkill (Sleep 3000 + retry) 被撤回后,
     NSIS 没有 belt-and-braces 兜底
2. 主席在 Settings 改角色绑定时, 默认 mimo 角色的下拉框是空
   → 即使 000096 修了 add_custom_provider, 第一次跑时
     `model_cache` 表为空, role picker 0 选项, 主席没法点保存

## 调查

### Bug A: 升级时 sidecar 文件被占用
- Tauri 2.x 自带的 `CheckIfAppIsRunning` 只检查主二进制
  (`flowntier-desktop.exe`)。当主席已关闭 GUI 但 sidecar
  `flowntier_runtime.exe` 仍在监听命名管道时, sidecar 持有
  文件 handle, `File /nonfatal` 写入失败 → NSIS 弹
  "无法打开要写入的文件", 只有 Abort / Retry / Ignore 三个按钮
- v0.4.21 (event 000061) 加了 v0.4.21 taskkill belt, 但
  当时 kill 没有 `/T` (进程树), 不能保证释放子进程句柄
- v0.4.22 (event 000092) 加了 v2 (Sleep 3000 + retry), 但
  主席报告整个系统冻结 → event 000093 撤回

### Bug B: 角色选择器在干净安装下空白
- `GET /api/router/models` 只回 `model_cache` 里有行的 provider
- 干净安装后 `model_cache` 为空 → 9 个 preset 全部没有行
  → `models: []` → Settings 的 `<select>` 一个 `<option>` 都没有
  → 主席无法保存 role_overrides (空 option 不能 select)
- 即使 000096 修了 `add_custom_provider`, 第一次跑时还没有
  任何 custom_provider 行 → 同样的空数组问题

## 修复

### Fix A: v3 NSIS taskkill belt
- `Function .onInit` 内:
  - `taskkill /F /IM flowntier-desktop.exe /T` (进程树)
  - `taskkill /F /IM flowntier_runtime.exe /T` (进程树)
  - `Sleep 1500` (kernel 释放文件句柄, 但不至于 freeze)
- `/T` 是关键: Tauri 把 sidecar spawn 成 desktop 的子进程,
  job-object 行为下 kill 父进程不一定级联, 必须显式 kill 进程树
- 1.5s 是 000093 (3s) 和 000061 (0s) 的折中: 够长释放 handle,
  不够长到 system freeze
- `patch-nsis.cjs` 标 marker `v0.4.22 taskkill-belt-bracess-v3`
  保证 idempotent (再次跑不会重复插入)
- **bug**: 旧版 patcher 写的代码模板 L127-140 有 floating string
  literals (`' ; closed the GUI...' ;` 后没加 `+` 接续), JS
  静默 no-op, 但语义混乱. 这次清理.

### Fix B: `/api/router/models` 多层 fallback
按优先级回填每个 provider 的至少一个 model:
1. `model_cache` 里的 `models_json` (live 拉过)
2. `OPENAI_FALLBACK_MODELS` (9 preset 的静态 catalog)
3. 对 custom provider: `c.default_model`
   (`thinking_strength:"medium"`, `context_length:null`,
   `source:"fallback"`)

这样保证角色选择器永远有至少一个可选项, 主席能立即保存
role_overrides, 之后 live fetch 异步刷新会替换 fallback 行.

### Fix C: GNU NSIS 重复块清理
之前 cargo tauri build --target x86_64-pc-windows-gnu 跑过一次
patcher 时, GNU installer.nsi 里 v0.4.21 legacy 块没被
`if (!content.includes('v0.4.22 ...'))` 短路掉 (因为它先
写入了 v3, 但前面 v0.4.21 标记也仍在). 手动删除冗余块,
保证两侧 installer.nsi 都只有 v3 一段.
