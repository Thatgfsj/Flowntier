# 000102 — patch-nsis.cjs 时机错了, build-with-patch.cjs 三阶段 build

## 主席反馈
"每次都是无法写入 O:\Flowntier\flowntier_runtime.exe"

升级 setup.exe 运行时, 即使 000097 的 v3 taskkill belt
代码存在于 `installer.nsi`, 实际编译出来的 setup.exe 里**没有**
对应的 taskkill 调用, 所以 NSIS 在 `File` 指令写入时被
sidecar 文件句柄锁住 → 弹"无法打开要写入的文件"。

## 调查

`scripts/patch-nsis.cjs` 通过 `build.beforeBundleCommand` 钩子
嵌入到 tauri build 流程. 文档说:

> "This will also run `build.beforeBundleCommand` before
> generating the bundles and installers of your app."

实测钩子跑得太早:

1. Tauri 跑 `beforeBundleCommand` ✓ (patcher 写入 v3 块)
2. Tauri bundler 渲染 `installer.nsi` (从 built-in template)
3. **Tauri 的渲染覆盖了我们的 patch** ← 关键 bug
4. makensis 编译没 patch 的 installer.nsi
5. 输出 setup.exe 没 taskkill

证据:
- 手动 `node scripts/patch-nsis.cjs` 后
  `target/release/nsis/x64/installer.nsi` L468-489 有 v3 block
- 跑 `tauri build` 后同一文件被覆盖 (size 31188 → 无 patch 内容)
- bundle/nsis/Flowntier_0.4.22_x64-setup.exe 也不含 taskkill

`grep -ac "taskkill" setup.exe` = 0 (字符串被 LZMA 压缩,
所以 0 不直接证明缺失, 但 7z l 看 setup.exe 是 NSIS-3 LZMA,
内部不含明文 taskkill, 跟 000099 setup.exe (`fbb82206...`)
size 8903206 vs 这次 `009f2459...` size 8905566 差 2.4KB ≈
v3 block 编码后体积).

## 修复

新脚本 `scripts/build-with-patch.cjs` 三阶段:

```
Phase 1: pnpm exec tauri build --no-bundle
  - 编译 binaries + sidecar
  - Tauri 生成 installer.nsi template (没 makensis)
  - beforeBundleCommand 跳过 (因为没 bundle)

Phase 2: node scripts/patch-nsis.cjs
  - 在 Tauri 生成的 installer.nsi 上打 v3 patch

Phase 3: 直接调 makensis 编译 patched installer.nsi
  - makensis 路径: %USERPROFILE%/AppData/Local/tauri/NSIS/makensis.exe
  - 编译产物 nsis-output.exe → 拷贝到 bundle/nsis/...
```

为什么这样 work:
- Phase 1 写完 installer.nsi 就停, 我们拿到的文件是最新 template
- Phase 2 直接改这个文件, 没人再覆盖
- Phase 3 单独跑 makensis, 输出到 Tauri 期望的 bundle 路径

## 文件改动

- `scripts/build-with-patch.cjs` 新增 (3-phase wrapper)
- `apps/desktop/package.json`:
  - `tauri:build` 改用 wrapper
  - 加 `tauri:build:raw` 保留旧的 raw 调用 (debug 用)
- `apps/desktop/src-tauri/tauri.conf.json`: 移除
  `beforeBundleCommand` (不再需要, wrapper 接管)

   ```
   taskkill /F /IM flowntier_runtime.exe
   taskkill /F /IM flowntier-desktop.exe
   ```
2. **装桌面新包** (`009f2459...`):
   - 升级现有 Flowntier 安装

3. **新建构建流程**:
   - 以后 `cd apps/desktop && pnpm tauri:build` 自动用 3-phase wrapper
   - 不用再手动 `pnpm tauri:patch`
