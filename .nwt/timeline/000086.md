# 000086 — 修复交付 (Fix A + Fix B)

## 修复 1: mimo preset base_url
**问题**：`crates/pipe-server/src/providers.rs:125` 写的是 `https://api.xiaomi.com/v1`，
DNS 不解析（NXDOMAIN）。

**实际探测**：
```
$ nslookup api.xiaomi.com      → Non-existent domain
$ nslookup api.xiaomimimo.com  → 111.13.213.63, 111.13.213.62
$ curl -X POST api.xiaomimimo.com/v1/chat/completions
  → {"error":{"message":"Invalid API Key","code":"401","type":"invalid_key"}}
```

**修复**：
```diff
-    base_url: "https://api.xiaomi.com/v1",
+    base_url: "https://api.xiaomimimo.com/v1",
```

**验证**：strings 检查新 sidecar 二进制：
```
https://api.xiaomimimo.com/v1  ✓
(没有 api.xiaomi.com)
```

## 修复 2: sidecar 启动时读 workdir.json
**问题**：`bin/flowntier-runtime.rs:27` 默认 `workspace = current_dir()`。
Tauri 启动 sidecar 时没传 `--workspace`，所以 runtime 用了桌面 app 所在目录
(`O:\Flowntier`)，而不是用户选的 `O:\try` (从 workdir.json 持久化)。

**修复**：sidecar 启动时检查 `<data_dir>/workdir.json`，优先用持久化路径。

**验证**：strings 检查：
```
v0.4.22 (event 000085): restored workspace from workdir.json  ✓
v0.4.22 (event 000085): workdir.json points at a non-directory; using cwd instead
v0.4.22 (event 000085): failed to parse workdir.json; using cwd
v0.4.22 (event 000085): failed to read workdir.json; using cwd
```
