## 完整逻辑验证 (NWT 000091 之后)

### 阶段 0: 启动
1. flowntier-runtime.exe 启动 → logs::init → workspace 恢复 (workdir.json) → 
   生成 bridge token → ServerState::new → 启动 RPC 16 workers + events 4 workers + 
   scheduler + HTTP bridge
2. Tauri shell 启动 → taskkill 旧 sidecar → spawn 新 sidecar → 探测 pipe

### 阶段 1: ChatZone 发送任务
3. ChatZone.tsx → `invoke('run_workflow', ...)` 
4. Tauri lib.rs:run_workflow → pipe_request (用 spawn_blocking, 60s timeout)
5. server.rs:sserve_rpc_connection 接收 (worker 0 第一次 first=true，之后 false)
6. dispatcher 找 handler
7. /api/run_workflow handler:
   - 创建 Orchestrator
   - 创建 CancellationToken，存 active_workflows[wf_id]
   - tokio::spawn(orch.run())
   - 返回 wf_id 立即给 Tauri

### 阶段 2: orchestrator 8 phases
8. run() → emit_phase("1-requirement") → run_agent(Chief)
9. run_agent(Chief):
   - build_candidates 收集 primary + fallback
   - 对每个候选 drive_single_agent
   - drive_single_agent 跑 agent.run(task) 订阅 events
   - status match 用 starts_with
10. agent loop 调 nwt_log / bash / read / write / patch / grep (全部已注册)
11. agent 完 → emit Done → orchestrator 收到
12. 进入 Phase 2-8

### 阶段 3: UI
13. events pipe → Tauri events_bridge → app.emit("wf:event", v) → React
14. useAgentStream filter by kind
15. broadcast buffer 8K 不再 Lagged

### 阶段 4: 取消
16. cancel_workflow Tauri cmd → /api/workflow/cancel body
17. handler 查 active_workflows → token.cancel()

### 阶段 5: 5h 边界
18. probe_provider_alive(role, model_id) → GET <base_url>/models
19. 2xx → clear_quota_failure
20. 4xx/5xx → mark_quota_rate_limited + nudge

### 阶段 6: 安全
21. HTTP bridge 接受带 FLOWNTIER_HTTP_BRIDGE_TOKEN 的请求
22. 任何 local process POST 127.0.0.1:8765/rpc 无 token → 401
