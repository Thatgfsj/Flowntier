# 000094 — 修 2 个致命 bug + UI 状态显示

## 主席反馈
1. "是不能用" — 完全不能跑
2. "页面没展示当前任务在哪: 主理/审核员/架构师 全部空闲 而且明明在工作还是空闲"

## 调查
读 `~/Desktop/Flwntier.log` (90KB, 23:25 启动)

### Bug #1 (致命): workflow 没真的跑
log 里只有 GET 请求 (`/api/kv/...`, `/api/rpc/version`,
`/api/sample/auth_login`, `/api/settings/secrets/seed`)
**0 个** `run_workflow` 请求。

**根因**: `lib.rs:start_workflow_cmd` 调用 `tauri_core::start_workflow()`
是 **stub** — 只写 DB row, 不调 orchestrator。所以 UI 发请求,
DB 有 `state=REQ_RECEIVED` 但根本没 agent 跑。

**修复**: `start_workflow_cmd` 改成调 `pipe_request("POST",
"/api/run_workflow", { task: text })` → 走 pipe-server 真正的
8-phase orchestrator (event 000068)。

### Bug #2 (致命): events pipe 永远 failed
log: `events pipe create failed; backing off ... 拒绝访问。
(os error 5)` 每秒重试, 永远失败。

**根因**: `worker 0` 用 `first_pipe_instance(true)` 创建 named
pipe — Windows 在 stale pipe 存在时拒绝返回 os error 5。

**修复**: 所有 worker 永远用 `first_pipe_instance(false)`。
Tokio 的 `ServerOptions::create` 已经会处理 primary creation, flag
只影响 second instance 语义。

### Bug #3 (UX): agent 卡片永远空闲
App.tsx applyEvent 处理 `transition` / `phase_transition` /
`milestone` 但**不**翻译 phase 名 → role, 所以没有角色卡片
翻成 "thinking"。

**修复**: 加 `phaseToRole()` mapper:
- requirement/plan/dispatch/repair/delivery → chief
- plan-review/final-review → critic-a (示意)
- develop → worker

phase_transition → 'active' 时 `setAgentStatus('thinking')`,
milestone 'completed' 时翻回 'idle'。
