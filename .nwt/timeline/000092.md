# 000092 — NSIS taskkill v2 (Sleep 3s + retry)

## 背景
主席装新安装包时弹窗 "无法打开要写入的文件
O:\Flowntier\flowntier_runtime.exe"。NSIS taskkill 跑了但
文件还是 locked。

## 静态分析
- 主席机器的 `O:\Flowntier\flowntier_runtime.exe` MD5 还是 
  `f69ccfd4` (Jul 15 01:04 的旧版)，证明**文件 copy 失败**
- Tauri shell (PID 47968) 和 sidecar (PID 44940) 都在跑
- NSIS patch (a)+(b)+(c) 都在 installer.nsi 里
- taskkill 用 `/T` flag 应该 kill child process

## 根因
Windows kernel 不会同步释放 file handles。
`taskkill /F` 标记进程为 killed，但 handle 关闭是异步的。
NSIS 紧跟着的 `File` 指令就 race against still-locked EXE。

## 修复
NSIS `.onInit` 在 taskkill 后加 `Sleep 3000` (3秒)
让 OS 真正释放 handle。再 `Sleep 1000` 后做第二次
taskkill pass 兜底（应对 UAC 拦截或 AV 拖延的情况）。

## 关于 Flwntier.log
chairman 说桌面 `Flwntier.log` 不见了。runtime 用 
`OpenOptions::create(true).append(true)` 写文件，**删了下次
启动会重建**。但主席没装上新版 (file copy 失败)，所以新 runtime
没跑起来 → log 不会出现。

装好新包后 log 会自动重建在 `~/Desktop/Flwntier.log`。
